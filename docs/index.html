Getting Started

Get up and running with gobacktrader in 5 minutes.

Installation

gobacktrader requires Go 1.21+. Install via go get:

terminal
go get github.com/thangtmkafi/gobacktrader

Your First Backtest

The minimum viable backtest needs: a data feed, a strategy, and Cerebro to run it.

main.go
package main

import (
    "fmt"
    "github.com/thangtmkafi/gobacktrader/engine"
    "github.com/thangtmkafi/gobacktrader/feeds"
)

// BuyAndHold buys on the first bar and holds forever.
type BuyAndHold struct {
    ctx    *engine.StrategyContext
    bought bool
}

func (s *BuyAndHold) Init(ctx *engine.StrategyContext) { s.ctx = ctx }

func (s *BuyAndHold) Next() {
    if !s.bought {
        // Position Sizing: Use all available cash!
        size := engine.AllInSizer{}.Size(s.ctx, s.ctx.Data())
        
        s.ctx.Buy(size)
        s.bought = true
    }
}

func main() {
    c := engine.NewCerebro()
    c.SetCash(100_000)
    c.AddCSV(feeds.DefaultYahooConfig("AAPL.csv"))
    c.AddStrategy(func() engine.Strategy { return &BuyAndHold{} })

    results, err := c.Run()
    if err != nil { panic(err) }

    fmt.Printf("Starting: $%.2f\n", results[0].StartingCash)
    fmt.Printf("Final:    $%.2f\n", results[0].FinalValue)
}
💡 Data Format DefaultYahooConfig expects Yahoo Finance CSV format: Date,Open,High,Low,Close,Adj Close,Volume

Writing a Strategy

A strategy implements the engine.Strategy interface:

strategy.go
type Strategy interface {
    Init(ctx *StrategyContext)  // called once before the loop
    Next()                      // called every bar
}

// Optional interfaces:
type NotifyOrderer interface {
    NotifyOrder(o *Order)       // called when order status changes
}
type NotifyTrader interface {
    NotifyTrade(t *Trade)       // called when a trade opens/closes
}

The StrategyContext gives you access to:

  • ctx.Data() — the live (bar-by-bar) DataSeries
  • ctx.PreloadedData() — all bars pre-loaded (for building indicators)
  • ctx.Buy(size), ctx.BuyBracket(size, takeProfit, stopLoss) — trading helpers
  • ctx.GetPosition(data) — current position for a data feed
  • ctx.GetCash(), ctx.GetValue() — portfolio state

Adding Indicators

Build indicators in Init() using the pre-loaded data series:

strategy.go
func (s *MyStrategy) Init(ctx *engine.StrategyContext) {
    s.ctx = ctx
    data := ctx.PreloadedData()  // all bars available
    s.sma  = indicators.NewSMA(data, 20)
    s.rsi  = indicators.NewRSI(data, 14)
    s.macd = indicators.NewMACD(data, 12, 26, 9)
}

func (s *MyStrategy) Next() {
    sma := s.sma.Line().Get(0)   // current bar's SMA
    rsi := s.rsi.Line().Get(0)   // current RSI
    if rsi < 30 && sma > 0 {
        s.ctx.Buy(10)
    }
}
📝 Why PreloadedData? Indicators pre-compute their entire output at construction time. ctx.Data() is empty during Init() (bars haven't been fed yet), so use ctx.PreloadedData() which has all bars already loaded.

Core Concepts

Understanding how gobacktrader works under the hood.

Architecture

gobacktrader follows the same high-level design as Python's Backtrader, adapted to Go idioms:

architecture
┌─────────────────────────────────────────────────┐
│                    Cerebro                       │
│                                                  │
│  ┌──────────┐   ┌──────────┐   ┌─────────────┐ │
│  │ DataFeed │──▶│  Broker  │──▶│  Strategy   │ │
│  │ CSV/WS/  │   │ Orders   │   │  Init()     │ │
│  │ NATS/... │   │ Positions│   │  Next()     │ │
│  └──────────┘   │ Trades   │   │  Notify*()  │ │
│       │         └──────────┘   └─────────────┘ │
│       ▼              │               ▲          │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   │
│  │Indicators│   │ Analyzers│   │  Sizers   │   │
│  │ SMA, RSI │   │ Sharpe   │   │ Fixed, %  │   │
│  │ MACD ... │   │ DrawDown │   │           │   │
│  └──────────┘   └──────────┘   └──────────┘   │
└─────────────────────────────────────────────────┘

Data Model

Two core types power all data access:

core.Line

A time-indexed float64 series with a cursor. Access values relative to the current bar:

  • line.Get(0) — current bar's value
  • line.Get(-1) — previous bar
  • line.Get(-n) — n bars ago

core.DataSeries

A collection of 7 named Lines: DateTime, Open, High, Low, Close, Volume, OpenInterest.

Data feeds populate the series bar-by-bar. Indicators read from the Close line (or any line).

Engine Flow

Here's what happens when you call c.Run():

  1. Load — all data feeds pre-read their data (CSV into memory)
  2. PreloadedData — a fully-populated DataSeries is built for indicator construction
  3. Init — each strategy's Init(ctx) is called (build indicators here)
  4. Main Loop — for each bar:
    • Feed advances (bar added to live DataSeries)
    • Broker processes pending orders against the new bar's prices
    • Equity snapshot is recorded to the equity curve
    • Order/trade notifications are delivered to strategies
    • Strategy Next() is called
  5. Analyzers — each analyzer's Analyze() is called with the final RunResult

Advanced Orders (Brackets & Trailing)

gobacktrader natively routes advanced OCO (One-Cancels-Other) groups to protect positions automatically.

usage
// Buy at market, place a limit take-profit at +5%, and stop-loss at -2%
price := ctx.Data().Close.Get(0)
entry, takeProfit, stopLoss := ctx.BuyBracket(size, price*1.05, price*0.98)

// Add trailing stops to lock in profits dynamically
ctx.Buy(size, engine.WithStopTrailPercent(0.05)) // Trails 5% behind highest high

Position Sizing

By default, orders take raw floating-point sizes, but gobacktrader provides utility modules to compute sizes against dynamic portfolio equity.

  • FixedSizer{Amount: 10}
  • PercentSizer{Percent: 0.10} — 10% of total portfolio value
  • AllInSizer{} — Uses 100% of available cash
usage
sizer := engine.PercentSizer{Percent: 0.10}
size := sizer.Size(ctx, ctx.Data())
ctx.Buy(size)

Indicators Reference

10 pre-built indicators covering trend, momentum, and volatility analysis.

Overview

All indicators follow the same pattern:

  1. Construct in Init() using ctx.PreloadedData()
  2. Read values in Next() via .Line().Get(ago)
  3. First period-1 bars return NaN (not enough data)

Trend Indicators

SMA — Simple Moving Average

Arithmetic mean of the last N closes. The most basic trend filter.

usage
sma := indicators.NewSMA(data, 20)
val := sma.Line().Get(0)  // current SMA(20)

EMA — Exponential Moving Average

Weighted average with exponential decay. k = 2/(period+1). Seeded with SMA of first period values.

usage
ema := indicators.NewEMA(data, 12)
// Also available: NewEMAOnLine(line, period) for chaining

WMA — Weighted Moving Average

Linearly weighted — recent bars have higher weights.

DEMA — Double EMA

Formula: 2×EMA - EMA(EMA). Reduced lag vs standard EMA.

TEMA — Triple EMA

Formula: 3×EMA - 3×EMA² + EMA³. Minimal lag.

Momentum / Oscillators

RSI — Relative Strength Index

Wilder smoothing, output clamped to [0, 100]. Standard period: 14.

usage
rsi := indicators.NewRSI(data, 14)
if rsi.Line().Get(0) < 30 {
    // oversold condition
}

MACD — Moving Average Convergence/Divergence

Three lines: MACD (fast EMA - slow EMA), Signal (EMA of MACD), Histogram (MACD - Signal).

usage
macd := indicators.NewMACD(data, 12, 26, 9)
macdLine := macd.Line().Get(0)
signal   := macd.Signal().Get(0)
hist     := macd.Histogram().Get(0)

Stochastic — Stochastic Oscillator

%K and %D lines. %K = (Close - LowestLow) / (HighestHigh - LowestLow) × 100.

usage
stoch := indicators.NewStochastic(data, 14, 3)
k := stoch.K().Get(0)   // %K line
d := stoch.D().Get(0)   // %D line (signal)

Volatility Indicators

ATR — Average True Range

True Range with Wilder smoothing. Measures volatility.

usage
atr := indicators.NewATR(data, 14)
stopLoss := data.Close.Get(0) - 2*atr.Line().Get(0)

BollingerBands

Mid (SMA), Upper (Mid + N×σ), Lower (Mid - N×σ). Includes BandWidth() and PercentB() helpers.

usage
bb := indicators.NewBollingerBands(data, 20, 2.0)
upper := bb.Upper().Get(0)
lower := bb.Lower().Get(0)
mid   := bb.Mid().Get(0)

Analyzers Reference

5 performance analyzers computed automatically after each run.

Overview

Analyzers are attached to Cerebro before Run(). They implement the engine.Analyzer interface:

interface
type Analyzer interface {
    Name() string
    Analyze(result *RunResult, datas []*core.DataSeries) error
    Print()
}

Returns

FieldDescription
TotalReturnPct(FinalValue - Start) / Start × 100
PnLAbsolute profit/loss in dollars
BestTradePnLHighest net PnL of any closed trade
WorstTradePnLLowest net PnL of any closed trade
AvgTradePnLAverage net PnL across all closed trades

Sharpe Ratio

Annualised Sharpe: (mean(R) - Rf) / stddev(R) × √AF

Config: RiskFreeRate (default 0), AnnualisationFactor (default 252 for daily).

Walks the equity curve bar-by-bar tracking peaks:

Live Data Feeds

Connect to real-time data sources for paper or live trading.

Architecture

Every live feed runs a background goroutine that pushes core.Bar values onto a Go channel. The engine's Next() blocks on that channel — so your strategy code is identical between backtest and live.

🚀 Native Live Routing gobacktrader fully supports LiveBroker implementations seamlessly swapping out the simulated engine. Your strategies use the exact same API to trade with real exchanges via custom OrderRouter integrations.

WebSocket Feed

websocket.go
feed := livefeeds.NewWebSocketFeed(livefeeds.WebSocketConfig{
    URL:            "wss://stream.binance.com:9443/ws/btcusdt@kline_1m",
    Symbol:         "BTCUSDT",
    ReconnectDelay: 5 * time.Second,  // auto-reconnect
    ParseFunc:      myBinanceParser,   // custom JSON parser
})

c.AddData(feed)
c.RunLive(ctx)

Brokers & Live Routing

The engine operates entirely on the BrokerBase interface. By default, it uses a simulated broker for backtesting, but it can be natively swapped for true paper/live trading.

Architecture

For live or paper trading, you can swap the simulated broker with a LiveBroker. The LiveBroker acts as a middleware adapter that honors the exact same simulated environment locally while concurrently routing real orders to an external exchange via an OrderRouter.

Custom Routers

Developing a custom broker integration involves simply implementing the 4-method OrderRouter interface:

order_router.go
type OrderRouter interface {
    // Submit a generic order to the exchange
    PlaceOrder(ctx context.Context, order *engine.Order) (string, error)

    // Cancel an order from the exchange using its unique ID
    CancelOrder(ctx context.Context, exchangeRef string) error

    // Reconcile live positions to the internal simulation state
    GetPositions(ctx context.Context) ([]livebrokers.LivePosition, error)

    // Reconcile live cash reserves
    GetCash(ctx context.Context) (float64, error)
}

Plugging It In

Initialize the LiveBroker with your router and delegate it to the Cerebro system.

live_runner.go
router := myexchange.NewRouter(apiKeys)

// Initialize LiveBroker with your router and starting cash
lb := livebrokers.NewLiveBroker(router, 100_000)

// Direct Cerebro to use the LiveBroker
c.SetBroker(lb)

// Profit!
c.RunLive(ctx)
